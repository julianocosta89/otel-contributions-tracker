# OTel Contributions Tracker

A static web app that visualizes [OpenTelemetry contribution data](https://insights.linuxfoundation.org/project/opentelemetry/contributors) sourced from the Linux Foundation Insights API. Built as native ES modules with no build step, designed to run on GitHub Pages with zero build tooling.

## Features

- **Overview** — stat cards for total contributors, organizations, maintainers, and reviewers (with % change vs. prior period), a contribution concentration chart, top organizations, and geographic reach
- **Contributors** — full leaderboard with avatars, GitHub handles, contribution counts, percentage share, and role badges; paginated 25 per page with search by name or handle
- **Organizations** — horizontal bar chart of the top 15 organizations plus a paginated leaderboard table
- **Concentration** — bus factor analysis showing how many top contributors account for what percentage of all work, with per-contributor and per-organization breakdowns
- **Geography** — country-level bar chart and table with flags, contribution counts, and share bars
- **Global filters** — date presets (30d, 90d, 6m, 1y), custom date range, platform filter, and activity type filter

### `index.html`

HTML skeleton only — structure, CDN script tags (Tailwind, Chart.js), and a single `<script type="module" src="js/main.js">`. No inline JavaScript. All UI changes go in the relevant module under `js/`; see [`js/README.md`](js/README.md) for which file owns what.

### `js/`

ES modules with no build step. [`js/README.md`](js/README.md) documents every module — its purpose, key exports, and how it relates to its neighbours. Tab modules are documented in [`js/tabs/README.md`](js/tabs/README.md); modal modules in [`js/modals/README.md`](js/modals/README.md).

### `css/`

A single `app.css` extracted from `index.html`. Tailwind covers almost everything; this file only adds the loading spinner, slim scrollbar, tab-button active state, modal slide-in transitions, role-badge tooltip positioning, and dark-mode overrides for native form controls. See [`css/README.md`](css/README.md).

### `docs/`

Standalone HTML pages that provide supplementary explanations for metrics and features used in the app. Each page is self-contained (same Tailwind CDN stack, dark theme to match the app) and linked from `index.html` at the point where users need more context than a tooltip can offer. See [`docs/README.md`](docs/README.md) for the page inventory and conventions for adding new pages.

### `scripts/`

Node.js 20+ ESM scripts that populate the `data/` directory. No `npm install` is needed — the scripts use only built-in Node.js modules and fetch. Run `fetch-affiliations.mjs` before `fetch-data.mjs` (attribution enrichment reads `affiliations.json` at startup), and both before `fetch-github-companies.mjs` (which fills gaps left by the other two). See [`scripts/README.md`](scripts/README.md) for per-script options and the recommended run order.

### `data/`

All files in this directory are generated by the scripts above, committed to the repository, and loaded by the app at startup. Committing generated data means the web app runs without any server-side logic — it reads static JSON files directly. The GitHub Actions workflow keeps these files up to date automatically. See [`data/README.md`](data/README.md) for a description of each file's schema.

## Getting Started

**Prerequisites:** Node.js 20+ (only needed to generate the cache; the web app itself has no dependencies).

1. Fork or clone the repository.

2. Fetch and cache the data (run in order — `fetch-data.mjs` reads `affiliations.json` at startup):

   ```bash
   make fetch-affiliations
   node scripts/fetch-data.mjs --full
   ```

   This creates `data/affiliations.json` (full gitdm history) and `data/cache.json` (LFX leaderboards with time-aware attribution).

3. Commit the generated data files and push:

   ```bash
   git add data/affiliations.json data/cache.json
   git commit -m "chore: seed initial data cache"
   git push
   ```

4. Enable GitHub Pages: go to **Settings → Pages** and set the source to deploy from branch `main`, root directory.

5. The included GitHub Actions workflow refreshes `data/cache.json` daily at 06:00 UTC and commits the update automatically. No further configuration is required.

## How It Works

### Data caching

The app loads `data/cache.json` at startup. When the active view matches the default settings (1-year window, all platforms, all activity types), the cached dataset is used directly — no network requests, no CORS issues, and search covers the full dataset.

Changing any filter triggers live API calls to the LF Insights endpoint. A badge in the app header indicates whether data is coming from the cache (with its date) or from a live request.

### Tech stack

- Vanilla JavaScript as native ES modules — no build step, no `npm install`, no bundler
- [Tailwind CSS](https://tailwindcss.com/) via CDN
- [Chart.js](https://www.chartjs.org/) via CDN
